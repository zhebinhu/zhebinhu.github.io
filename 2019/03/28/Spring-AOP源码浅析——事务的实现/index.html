Spring AOP 源码浅析——事务的实现

2019-03-28 Java, Spring 0 评论

Spring AOP 的诸多应用中，事务无疑是最常使用的工具之一。对于 Spring 事务的有些特性我们很熟悉，比如事务的四大特性 ACID，比如具体的实现委托给底层 DB。但有些特性我又有些陌生，比如事务传播机制，事务实现的原理。这也是这篇文章要阐述的内容。

一、事务的七大传播机制

如上，当我们在其它地方调用开启了事务的 serviceA() 和 serviceB() 时，就发生了事务的传播。这里面会有很多种情况，比如调用方当前存不存在事务，如果存在事务的话，是要加入当前的事务还是自己创建一个；如果不存在的话，是自己创建一个还是抛异常。这些策略的选择，统称为 Spring 的事务传播机制。

org.springframework.transaction.annotation 包下的枚举类 Propagation 为 Spring 定义了七种传播机制，我们简单了解一下：

事务传播机制 说明
REQUIRED 如果当前已经存在事务，那么加入该事务，如果不存在事务，创建一个事务，这是默认的传播属性值。
SUPPORTS 如果当前已经存在事务，那么加入该事务，否则在无事务环境执行。
MANDATORY 当前必须存在一个事务，否则抛出异常。
REQUIRES_NEW 暂停当前事务(当前无事务则不需要)，创建一个新事务。
NOT_SUPPORTED 暂停当前事务，在无事务环境上执行。
NEVER 如果当前存在事务，则抛出异常，否则在无事务环境上执行。
NESTED 如果当前存在事务，则嵌套事务执行
